01 - 汇编入门

1.汇编语言

汇编语言：https://www.youtube.com/watch?v=75gBFiFtAb8&themeRefresh=1

汇编代码是由处理器在计算机上实际运行的代码。汇编代码有很多不同的架构。不同类型的处理器可以运行不同类型的汇编代码。最常接触的两种类型是 64 位和 32 位 ELF（可执行与可链接格式）。通常会把这两种分别称为x64和x86。（64位和32位）

2.寄存器

在x64的 Linux 系统中，函数的参数是通过寄存器传递的。前几个参数通过以下寄存器传递：

rdi：第一个参数
rsi：第二个参数
rdx：第三个参数
rcx：第四个参数
r8：第五个参数
r9：第六个参数

在x86的 ELF 架构中，参数是通过栈传递的。另外，如你所知，在 C 语言中函数可以返回一个值。在x64中，这个返回值存放在rax寄存器中。在x86中，则存放在eax寄存器中。

寄存器也有不同的大小。我们通常会接触到的大小有 8 字节、4 字节、2 字节和 1 字节。存在这些不同大小的寄存器是技术发展的结果，这样我们就能在寄存器中存储更多的数据。

+-----------------+---------------+---------------+------------+
|  8字节寄存器      |    低4字节     |     低2字节    |    低字节   |
+-----------------+---------------+---------------+------------+
|   rbp           |     ebp       |     bp        |     bpl    |
|   rsp           |     esp       |     sp        |     spl    |
|   rip           |     eip       |               |            |
|   rax           |     eax       |     ax        |     al     |
|   rbx           |     ebx       |     bx        |     bl     |
|   rcx           |     ecx       |     cx        |     cl     |
|   rdx           |     edx       |     dx        |     dl     |
|   rsi           |     esi       |     si        |     sil    |
|   rdi           |     edi       |     di        |     dil    |
|   r8            |     r8d       |     r8w       |     r8b    |
|   r9            |     r9d       |     r9w       |     r9b    |
|   r10           |     r10d      |     r10w      |     r10b   |
|   r11           |     r11d      |     r11w      |     r11b   |
|   r12           |     r12d      |     r12w      |     r12b   |
|   r13           |     r13d      |     r13w      |     r13b   |
|   r14           |     r14d      |     r14w      |     r14b   |
|   r15           |     r15d      |     r15w      |     r15b   |
+-----------------+---------------+---------------+------------+

在x64中我们会见到 8 字节的寄存器。然而在x86中，我们能使用的最大寄存器是像ebp、esp、eip等 4 字节的寄存器。当然，我们也可以使用比架构最大寄存器尺寸更小的寄存器。

在x64中有rax、eax、ax和al寄存器。rax寄存器是完整的 8 字节。eax寄存器只是rax寄存器的低 4 字节。ax寄存器是rax寄存器的最后 2 字节。最后，al寄存器是rax寄存器的最后 1 字节。

3.数据存储单位

  • 位（bit）：是计算机数据存储的最小单位，只有 0 和 1 两种状态。8 位组成 1 字节，例如在描述二进制数据时，会精确到每一位的状态。像一个字节数据01010101，这里面就包含了 8 个位。
  • 字节（Byte）：是最基本的存储单位，1 字节等于 8 位。在计算机中，文件大小、内存占用等常用字节来衡量，比如一个简单的文本文件大小可能是几千字节。ASCII 码中一个字符就占用 1 字节空间，像字符 ‘a’ 在内存中存储就占 1 字节。
  • 字（Word）：通常表示 16 位（2 字节）的数据存储单位。在早期 16 位的计算机系统中应用广泛，比如 8086 处理器，其通用寄存器 AX、BX 等可以存放一个字的数据。在一些编程语言中，像汇编语言，使用 “字” 能更精准地操作内存和寄存器数据，可进行 16 位数据的算术运算、逻辑运算等。
  • 双字（Double Word，dword（double word））：代表 32 位（4 字节）的数据存储单位。==在 32 位计算机系统，如 x86 架构中应用广泛。==像 32 位的整数类型（如 C 语言中的int在 32 位系统下通常是 4 字节）就占据一个双字空间。在处理较大整数运算、内存地址表示（32 位系统下内存地址为 4 字节）等场景时经常使用，保证数据的完整性和运算准确性。
  • 四字（Quad Word，qword（quadra word））：指 64 位（8 字节）的数据存储单位。==在 64 位计算机系统中常见，像 x86 - 64 架构，其通用寄存器如 RAX、RBX 等可以存放四字数据。==在处理高精度计算（如科学计算中的大整数运算）、64 位内存地址访问等场景中发挥重要作用，64 位的操作系统和软件利用四字数据单位提升数据处理效率和系统性能。

所以呀，1 个“四字”等于 2 个 “双字”，等于 4 个 “字”，等于 8 个“字节”，等于 64 “位”。

也就是：1qd = 2dd = 4wd = 8B = 64b

02 - 工具入门

1.ida的使用

  • 跳转到入口点：c

  • 查看段：Ctrl+S

  • 查看程序中的函数（默认）：Shift+F3

  • 看结构体：Shift+F9

  • 添加标签：按Alt+M组合键可添加标签；通过Ctrl+M快速跳转到标记处

  • 查找： Alt+T或Ctrl+F

  • 切换数据存储格式：右键变量。

    或者：

    Q:十六进制

    H:十进制

    R: 字符

    A:普通字符串

    Alt+D:选择Q word或者TByte?

    Alt+A:字符串

    选中，按C，标记为code

    选中，按U，取消
